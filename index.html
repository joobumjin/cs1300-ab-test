During this project, students were given an appointment scheduling interface (Version A). Our goal was to redesign the interface (Version B) and analyze the gathered data to test whether our redesigned interfaces improved the interface in tangible ways.

This project utilized A/B testing techniques to analyze how the user experience changed from Version A to Version B and draw conclusions about how Version B lead to different user experiences.

The change I primarily focused on was an intuitive ordering of appointments.

In Version A, the appointments seemed to not have any specific order.

In Version B, I decided to sort the appointments first by location and then by time. Additionally, the buttons darken on hover to look more intuitively clickable.

Quanitfying Changes in UX

What metrics can we use to quanity the user experience and how it might change?

Throughout this project, I focused on 3 primary metrics:

1.

How many users misclicked?

2.

How long did users take to find the right button?

3.

How long did users take to click on any button?

1. Misclick Rate

How many users clicked a different button on the page before finding the correct button for the task?

Null Hypothesis

There is no significant difference between the number of Ver A users who misclicked and Ver B users who misclicked.

Alternative Hypothesis

The number of Ver B users who misclicked is significantly less than that of Ver A users.

I predicted that the collected data would support the alternative hypothesis, because the changes made for Ver B hopefully clarified page navigation, decreasing the chance that someone clicks on a wrong button.

2. Time Spent on Page

Time spent on the webpage for each user group

Null Hypothesis

There is no significant difference between the time users spent on Ver A and the time spent on Ver B.

Alternative Hypothesis

The time users spent on Ver B is significantly less than the time spent on Ver A.

I predicted that the collected data will support the alternative hypothesis, because the changes for Ver B hopefully made page navigation easier, allowing users to find the correct appointment faster.

3. Time Until First Click

This will be defined as the time before the user's first click. I thought this would be meaningful to observe because the time until first click might be correlated with the users' confidence in their ability to complete their task.

Null Hypothesis

There is no significant difference between the time before Ver A users' first clicks and the time before Ver B users first clicks.

Alternative Hypothesis

The time before a Ver B user's first click is significantly less than the time before a Ver A user's first click.

I predicted that the collected data will support the alternative hypothesis. I thought that the changes I made for Ver B would both make button interaction more intuitive and make finding the relevant appointment faster.

Data & Analysis

1. Misclick Rate

The data for this experiment is categorical, with each user for each version being categorized as TRUE (did misclick), or FALSE (did not misclick).

I used the Chi-Square Test to determine the probability of the TRUE/FALSE distribution of Version A users and Version B users being the same.

The Chi-Square Test returned a p-value of 4.28%, meaning that there is a 4.28% probability that the distribution of Ver A and Ver B users are the same. Given this is less than 5%, we can conclude that this is statistically significant

The data seems to support that the misclick distribution of Version A users is not the same as the misclick distribution of Version B users.
